Security Architecture

Multi-Layer Protection

🔐

Data Encryption

  • In Transit: TLS 1.3 encryption for all data transmitted between your browser and our servers. HSTS enabled to prevent downgrade attacks.
  • At Rest: AES-256 encryption for all stored data including user profiles, financial records, chat history, and uploaded documents.
  • Key Management: Encryption keys are managed through industry-standard key management systems with automatic rotation every 90 days.
  • API Security: All API endpoints secured with JWT tokens, rate limiting, and request signature validation.
🐀

Database Security

  • Row Level Security (RLS): PostgreSQL RLS policies ensure users can only access their own data — even if there's an application-level vulnerability.
  • Supabase Infrastructure: Our database is hosted on Supabase with SOC 2 Type II certified infrastructure, automatic backups, and point-in-time recovery.
  • Query Parameterization: All database queries use parameterized statements, preventing SQL injection attacks by design.
  • Access Control: Principle of least privilege — each service component has only the minimum database permissions required for its function.
🤖

AI & Privacy

  • No Training on Your Data: Your conversations with our AI advisor are never used to train AI models. Your financial queries remain private.
  • Secure API Channels: AI requests are processed through encrypted API channels to OpenAI/Google/Anthropic with data processing agreements in place.
  • Data Minimization: We send only the minimum context necessary for AI processing — never your complete financial profile.
  • Chat History Control: You can delete your chat history at any time. Deleted conversations are permanently removed within 24 hours.
💻

Application Security

  • Authentication: Secure email/password with bcrypt hashing (saltRounds: 12), OAuth 2.0 social login, and optional two-factor authentication.
  • Session Management: HTTP-only, Secure, SameSite cookies with automatic session expiry and re-authentication for sensitive operations.
  • CSRF Protection: Cross-Site Request Forgery tokens on all forms and state-changing API endpoints.
  • Input Validation: Server-side validation on all inputs with content security policies preventing XSS attacks.

Your Safety

Protect Yourself Too

Security is a partnership. Here are steps you can take to protect your account:

01

Use Strong Passwords

Minimum 12 characters, mix of uppercase, lowercase, numbers, and symbols. Never reuse passwords across sites. Use a password manager.

02

Enable 2FA

Add two-factor authentication to your account for an extra layer of security. Even if your password is compromised, your account stays safe.

03

Verify Communications

We will never ask for your password via email or phone. All official emails come from @finucity.com. Report suspicious messages.

04

Keep Software Updated

Ensure your browser and operating system are up-to-date. Security patches close vulnerabilities that attackers exploit.

Found a Vulnerability?

We appreciate responsible disclosure. If you've found a security issue, please report it to us. We take every report seriously and will respond within 24 hours.
